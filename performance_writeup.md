## Fake Data Modeling
[link to fake.py](https://github.com/SumedhaKun/performance-reviews-api/blob/main/fake.py)  \
We generated 120,000 employees, 204 companies, ~1,800,000 appraisals, ~1,680,000 performance reviews, and 2,400,000 comments.
These companies were split into 60 mini companies (with 60 employees), 122 small companies (with 200 employees), 16 medium companies (with 2000 employees), and 6 large companies (with 10000 employees). Due to the fact that there are less larger companies in the world, we thought there would be less larger companies using our service. However, the mini companies would have less of a reason to use our service. The attributes for the companies were simply generated using Faker and choosing randomly from our pre-set list of industries.

We used Faker and the random module to generate our 120,000 employees. Employee hire date was randomly generated from between now and 30 years ago. When generating each employee, company, title_id, and level are left blank to be filled in later. To assign each employee to a company, we go through the array in sections that are based off the predefined company sizes. Once an employee is assigned to a company in this way, we can fill in company, title_id, and level based off their position in the array. For example, the first employee in the company's segment of the array gets the CEO title and the highest level. The CEO is also guaranteed to be an active employee, with a department of 'All', as opposed to the rest of the employees, where those traits are random. As we go through each employee, we also generate performance reviews, appraisals, and comments.

Appraisals and performance reviews were generated pretty similarly. As we went through the employees, one PR and one appraisal was generated for each year that they had been working at the company. This meant an average of ~15 appraisals and ~14 performance reviews per employee, which was verified by experimentation. The text in the appraisals and PRs are randomly generated using Faker, with 1-3 sentences. Additionally, the scores on a PR are generated by sampling randomly using numpy from a normal distribution centered at 7 with a standard deviation of 1. The reviewer ID is selected randomly from the segment of the company that contains managers.

Finally, each employee gets 20 generated comments. We chose to do it this way because comments aren't strictly related to years at the company, unlike appraisals and performance reviews. The comment text is generated using Faker, and the subject is randomly selected from a list. The commenter ID is selected randomly from all employees in the company.

As stated above, we generated 120,000 employees, 204 companies, ~1,800,000 appraisals, ~1,680,000 performance reviews, and 2,400,000 comments, which adds up to ~6,204,000 rows. Since each employee has spent an average of 15 years at each company, that means 120,000 _ ~15 = ~1,800,000 appraisals. Similarly, 120,000 _ ~14 = ~1,680,000 performance reviews. 20 rows of comments per employee means 2,400,000 rows of comments total. In total, we generated a little over six million rows of data, with some leeway for random variation.

## Query plans

### GET /employees/{employee_id}/stats

```
EXPLAIN
SELECT
	COUNT(id) AS total_reviews,
	AVG(category_1) AS average_category_1,
	AVG(category_2) AS average_category_2,
	AVG(category_3) AS average_category_3,
	COUNT(*) FILTER (
		WHERE
			title_change = 1
	) AS title_change_count,
	COUNT(*) FILTER (
		WHERE
			level_change = 1
	) AS level_change_count
FROM
	performance_reviews
WHERE
	employee_id = 25
```

Before:

```
Finalize Aggregate (cost=38251.59..38251.60 rows=1 width=120)
-> Gather (cost=38251.34..38251.55 rows=2 width=120)
Workers Planned: 2
-> Partial Aggregate (cost=37251.34..37251.35 rows=1 width=120)
-> Parallel Seq Scan on performance_reviews (cost=0.00..37251.18 rows=8 width=24)
Filter: (employee_id = 25)
```

It does a sequential scan on performance reviews looking for the employee_id.

Added index for employee_id:

```
CREATE INDEX stats_employee_id_index
ON performance_reviews (employee_id);
```

After:

```
Aggregate (cost=9.19..9.20 rows=1 width=120)
-> Index Scan using stats_employee_id_index on performance_reviews (cost=0.43..8.78 rows=20 width=24)
Index Cond: (employee_id = 25)
```

Now takes < 30 ms
